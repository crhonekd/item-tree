# Phase 0 — Scaffolding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up an empty but runnable Spring Boot service for the ITEMTREE project on Phase A constraints — H2 (Oracle compat mode), public-repo dependencies only, `dev` profile default — with the agreed package layout in place and a smoke test that proves the context loads and the seed data is present.

**Architecture:** Single Gradle (Kotlin DSL) module. All app code under `com.myxcomp.ice.xtree`. `dev` is the default and only active profile in Phase A. Schema and seed data load through Spring's `spring.sql.init` mechanism against an in-memory H2 instance in `MODE=Oracle`. Package skeleton scaffolded with empty `package-info.java` files so later phases just add classes. No features are implemented in this phase — only the foundation.

**Tech Stack:** Java 21, Spring Boot 3.4.x, Spring MVC, Spring `JdbcClient` + HikariCP (transitive), H2 (`MODE=Oracle`), Jackson (`jsr310`, XML mapper), Lombok, Micrometer + Prometheus registry, SpringDoc OpenAPI starter, JUnit 5 + AssertJ + Mockito (via `spring-boot-starter-test`).

**Phase boundaries:** This plan covers ONLY Phase 0 from `IMPLEMENTATION_NOTES.md`. Do not anticipate Phase 1 work (no OpenAPI spec, no generated controllers). Empty `package-info.java` files are fine; classes inside them are for later phases.

## Task 3: Database schema

**Files:**
- Create: `src/main/resources/db/schema.sql`

- [ ] **Step 1: Write `src/main/resources/db/schema.sql`**

```sql
-- ITEMTREE — production schema mirror.
-- H2 in MODE=Oracle accepts these types and the recursive CTE / sequence syntax.
CREATE TABLE ITEMTREE (
    ITEMTREEID     NUMBER(10) PRIMARY KEY,
    PARENTID       NUMBER(10) NOT NULL,
    NAME           VARCHAR2(70) NOT NULL,
    TYPE           VARCHAR2(30) NOT NULL,
    XML            CLOB,
    LASTUPDATEUSER VARCHAR2(20),
    LASTUPDATE     DATE,
    JSON           CLOB
);

CREATE INDEX IDX_ITEMTREE_PARENTID  ON ITEMTREE(PARENTID);
CREATE INDEX IDX_ITEMTREE_LASTUPDATE ON ITEMTREE(LASTUPDATE);

CREATE SEQUENCE ITEMTREE_ID_SQN START WITH 100000 INCREMENT BY 1;
```

**Notes:**
- Column order matches production §2.
- Sequence starts at `100000` so the hand-coded seed ids in `data.sql` (1–99) never collide with sequence-generated ids in later phases.
- `LASTUPDATE` is a `DATE` column by spec; UTC convention is enforced at the JDBC boundary in later phases by `TimeMapper` (Phase 2).
